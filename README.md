ЗАДАЧА 1.

Выбор пал на технологию Module Federation так как она обладает рядом важных в данном случае моментов:

- Динамическая загрузка модулей. Module Federation позволяет динамически загружать модули в браузер, что способствует уменьшению времени загрузки и повышению производительности приложения.
- Независимость развертывания. Используя Module Federation, можно разрабатывать и развертывать различные части приложения (например, аутентификация, управление контентом или профиль) независимо друг от друга.
- Повышенная модульность. Каждая часть системы может управляться и обновляться независимо.
- Постепенный переход к микросервисам. Если приложение изначально монолитное, использование Module Federation позволяет постепенно переводить части системы на микросервисную архитектуру без необходимости вносить глобальные изменения сразу.

текущий проект разделил на 4 модуля
    - auth работа с логином и авторизацией
    - content работа с фотографиями
    - profile работа с профилем
    - host все взаимодействие с микросервисами
